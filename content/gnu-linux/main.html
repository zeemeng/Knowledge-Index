About the GNU Project. About GNU software. About GNU Hurd. About Linux-libre.

GNU refers primarily to two concepts. The first being a mass collaboration, free and open-source software project started by Richard Stallman in 1983 and the second being the vast collection of free software maintained by the said project.

GNU software notably includes an extensive set of low-level programs that may be used on top of an operation system kernel such as Linux or a BSD variant to produce a "full-fledged" and "ready-to-use" operating system. GNU software also notably includes tools that may be used to compile and build other software from source. Among such essential GNU programs are the GNU Compiler Collection (GCC), the GNU C library (glibc), GNU Core Utilities (coreutils), the GNU Debugger (GDB), GNU Binary Utilities (binutils) and the GNU Bash shell.

The GNU Project also maintains two kernels itself. The first, called GNU Hurd, is an original kernel that is developped in-house and which uses a microkernel architecture. Since it's inception, GNU Hurd has been subject to a slow pace of development with intermitent periods of stagnation. As such, it is generally considered less suitable for production than alternatives such as Linux and BSDs. The second, called Linux-libre, "is a modified version of the Linux kernel that contains no binary blobs, obfuscated code, or code under proprietary licenses. Binary blobs are software components with no available source code. In the Linux Kernel, they are mostly used for proprietary firmware images. While generally redistributable, binary blobs do not give the user the freedom to audit, modify or, consequently, redistribute their modified versions. The GNU Project attempts to keep Linux-libre in synchronization with the mainline Linux kernel."

About the Linux kernel.

Linux refers to a free and open-source operating system kernel that is created by Linus Torvalds in 1991. Linux is production ready and has been under continuous and active maintenance and development to this day. (2021 at the time of writing)

GNU/Linux refers to any "full-fledged" and "ready to use" operating system comprised of the Linux kernel in addition to a set of essential, low-level GNU programs. Such a system may optionally include other layers of higher-level programs such as a display server, a window manager, a desktop environment and etc.

Discussion about Nvidia's dubious practice of locking up their graphics cards' power management and reclocking features. About Nvidia's use of signed firmware.

nouveau is an open-source driver for Nvidia graphics cards. It is an alternative to Nvidia's proprietary drivers. However, its performance is often far worse than that of the proprietary ones. Some online sources have pointed out that this is due to Nvidia's use of signed firmwares for their cards which restrict power management and reclocking features to only work with their proprietary drivers. This means that nouveau is only able to operate Nvidia cards at the lowest consumption state.

Arch Wiki - Guide on using QEMU. Article about using virt-manager to manage virtual machines. KVM, QEMU, and virt-manager constitute the main components of an efficient and performant virtualization solution for Linux host machines. Kernel-based Virtual Machine, or KVM, is a low-level virtual machine hypervisor that operates as a loadable Linux kernel module and provides the core virtualization infrastructure in this stack. QEMU integrates with KVM and provides machine and device emulation. Finally, virt-manager is a user friendly GUI front-end for the open-source libvirt API that is designed as a common interface/solution to manage various hypervisors and virtualization platforms, including KVM/QEMU. When using the APT package manager to install the different components of this virtualization solution, it is sufficient to simply install virt-manager (i.e. apt install virt-manager), as the other components will be automatically pulled in as dependencies. After all the components have been installed, it is important to add relevent users to the libvirt user group to allow them to manage virtual machines.
